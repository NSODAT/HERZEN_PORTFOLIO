# План реализации веб-портфолио (GitHub Pages)

## Цель

Одностраничное портфолио студента ИИТ с разделами **Обо мне**, **Проекты**, **Контакты**, удобным просмотром PDF и ссылками на Google Диск/Документы.

## Архитектура

| Компонент | Назначение |
|-----------|------------|
| `index.html` | Разметка секций и модального окна просмотра PDF |
| `css/main.css` | Стили, адаптив, тёмная тема |
| `js/app.js` | Загрузка `data/portfolio.json`, рендер карточек, модальное окно |
| `data/portfolio.json` | **Единственный файл для редактирования контента** |
| `assets/pdfs/` | PDF-файлы, загружаемые в репозиторий |
| `assets/img/` | Фото, иконки |

Статический сайт без сборки — подходит для GitHub Pages из ветки `main` / папки root.

## Разделы страницы

1. **Hero** — имя, специальность, краткий слоган, навигация по якорям.
2. **Обо мне** — биография, институт, курс, навыки (из JSON).
3. **Проекты** — категории с фильтром по типу:
   - дипломная работа;
   - курсовые;
   - практики;
   - лабораторные.
4. **Контакты** — email, Telegram, GitHub, телефон.

## Модель данных проекта

В `portfolio.json` каждая работа описывается полями:

```json
{
  "title": "Название",
  "semester": "7 семестр",
  "description": "Кратко о работе",
  "tags": ["Python", "отчёт"],
  "link": {
    "type": "pdf | google-drive | google-doc | external",
    "url": "assets/pdfs/otchet.pdf"
  }
}
```

### Типы ссылок

| type | Поведение на сайте |
|------|-------------------|
| `pdf` | Открытие во встроенном просмотрщике (модальное окно) + кнопка «Скачать» |
| `google-drive` | Превью в модальном окне (`/preview`) или новая вкладка для папок |
| `google-doc` | Превью Google Docs в iframe |
| `external` | Переход по ссылке (GitHub, сайт и т.д.) |

## Быстрое наполнение контентом

1. Открыть `data/portfolio.json`.
2. Изменить блок `profile`, `contacts`.
3. Добавить работу в нужную категорию в `categories[].projects[]`.
4. Для PDF: положить файл в `assets/pdfs/` и указать путь в `link.url`.
5. Закоммитить и запушить — GitHub Pages обновится за 1–3 минуты.

Подробная инструкция — в `README.md`.

## Публикация на GitHub Pages

1. Создать репозиторий на GitHub (например `username.github.io` или `portfolio`).
2. Скопировать содержимое папки `portfolio` в репозиторий.
3. **Settings → Pages → Source**: Deploy from branch `main`, folder `/ (root)`.
4. Сайт: `https://<username>.github.io/<repo>/` (для user pages — без имени репозитория).

## Отличия от примера [Buryakov portfolio](https://buryackov-ivan.github.io/Buryakov_I_O_portfolio_2020-2024/projects.html)

- Одна страница вместо нескольких HTML.
- Контент в JSON, без правки разметки при добавлении работ.
- Встроенный просмотр PDF без ухода со страницы.
- Фильтр по типам работ (диплом, курсовая, практика, лабораторная).
